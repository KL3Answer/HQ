Elasticsearch 时序数据性能测试

2019-09-02 • 标签: 笔记

Elasticsearch

  1. 2 billion docs per shard at most
  2. 手动清除过期数据(index)
  3. 会产生大量的index 和 shard
  4. 记录 index 分派策略为 INDEX_PATTERN_NAME{pk}_{yyyyMMdd}，其中pk为当前记录所属的产品的productKey，时间为当前的记录中的日期
